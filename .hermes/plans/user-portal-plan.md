# 员工自助门户 + RBAC 权限体系 实施计划

## 一、数据库层 — 新增角色表

| 表 | 说明 |
|---|---|
| `sys_role` | 角色定义（admin / employee） |
| `sys_user_role` | 用户-角色关联 |
| `sys_role_menu` | 角色-菜单权限关联 |

admin 角色拥有所有菜单权限，employee 角色只拥有自助门户相关菜单。

## 二、后端 — 新增 API

| 模块 | API |
|---|---|
| 角色管理 | CRUD: `/system/role` 增删改查 |
| 用户角色 | 给用户分配角色 |
| 角色菜单 | 角色绑定菜单权限 |
| 菜单过滤 | `/system/menu/tree` 根据角色过滤可见菜单 |
| 用户信息 | `/auth/userinfo` 返回真实 `roles`+`permissions` |

## 三、前端管理端新增

- **角色管理页面** — 在「系统管理」下新增「角色管理」，支持创建角色、分配菜单权限
- **用户角色分配** — 在用户管理（如有）或角色管理中加入

## 四、前端用户端（新建）

员工登录后跳转到用户端门户，包含：

| 页面 | 说明 |
|---|---|
| 首页面板 | 个人统计：待办、考勤概况、工资概览 |
| 个人信息 | 查看/编辑个人资料 |
| 我的考勤 | 查看考勤明细 |
| 我的请假 | 提交请假、查看进度 |
| 工资条 | 查看历史工资记录 |
| 我的绩效 | 查看绩效结果 |
| 我的培训 | 查看培训安排 |
| 我的合同 | 查看合同信息 |

用户端使用**简化布局**（Top导航 + 内容区），不是管理端的左侧菜单布局。

## 五、路由体系

- `/?role=admin` → 跳到管理端（现有）
- `/?role=employee` → 跳到用户端（新建）
- 登录后根据角色自动跳转
